## Технологии

- **Express.js** — веб-фреймворк
- **MongoDB** — база данных
- **Monk** — MongoDB ODM
- **Joi** — валидация данных
- **Helmet** — защита заголовков HTTP
- **Morgan** — логирование запросов
- **Dotenv** — управление переменными окружения

---

## Установка и запуск

```bash
# Клонировать репозиторий
git clone https://github.com/Nihaochingiz/express-api.git

# Перейти в директорию проекта
cd express-api

# Установить зависимости
npm install

# Создать файл .env и настроить переменные (см. ниже)

# Запустить в режиме разработки
npm run dev

# Запустить в production
npm start
```

---

## Переменные окружения

Создайте файл `.env` в корне проекта:

```env
# Порт сервера (по умолчанию 8080)
PORT=8080

# URL подключения к MongoDB
DB_URL=mongodb://localhost:27017/employees

# URL для тестовой базы данных (опционально)
TEST_DB_URL=mongodb://localhost:27017/employees_test
```

## API Endpoints

Базовый URL: `http://localhost:8080`

### GET / - Получить всех сотрудников

Возвращает массив всех сотрудников из базы данных.

**Пример ответа:**

```json
[
  {
    "_id": "60d5f9f8b8e5a72d4c8e4567",
    "name": "Иван Петров",
    "job": "Разработчик"
  },
  {
    "_id": "60d5f9f8b8e5a72d4c8e4568",
    "name": "Мария Сидорова",
    "job": "Дизайнер"
  }
]
```

---

### GET /:id - Получить сотрудника по ID

Возвращает данные конкретного сотрудника.

**Параметры URL:**

| Параметр | Тип | Описание |
|----------|-----|----------|
| id | string | MongoDB ObjectId сотрудника |

**Пример запроса:**

```
GET /60d5f9f8b8e5a72d4c8e4567
```

**Пример ответа (успех):**

```json
{
  "_id": "60d5f9f8b8e5a72d4c8e4567",
  "name": "Иван Петров",
  "job": "Разработчик"
}
```

**Ошибка (сотрудник не найден):**

```json
{
  "message": "Employee does not exist",
  "stack": "..."
}
```

---

### POST / - Создать нового сотрудника

Создает нового сотрудника в базе данных.

**Тело запроса (JSON):**

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| name | string | Да | Имя сотрудника (минимум 3 символа, максимум 30) |
| job | string | Да | Должность сотрудника (минимум 3 символа, максимум 30) |

**Пример запроса:**

```json
{
  "name": "Алексей Смирнов",
  "job": "Team Lead"
}
```

**Пример ответа (успех, статус 201):**

```json
{
  "_id": "60d5f9f8b8e5a72d4c8e4569",
  "name": "Алексей Смирнов",
  "job": "Team Lead"
}
```

**Ошибка (сотрудник уже существует):**

```json
{
  "message": "Employee already exists",
  "stack": "..."
}
```

**Ошибка (некорректные данные):**

```json
{
  "message": "\"name\" length must be at least 3 characters long",
  "stack": "..."
}
```

Обновляет данные существующего сотрудника.

**Параметры URL:**

| Параметр | Тип | Описание |
|----------|-----|----------|

```

```

**Пример ответа:**

```json
{
  "_id": "60d5f9f8b8e5a72d4c8e4567",
  "name": "Иван Петров",
  "job": "Senior Разработчик"
}
```

## Обработка ошибок

API использует централизованную обработку ошибок через middleware.

**Коды статусов:**

| Код | Описание |
|-----|----------|
| 200 | Успешный запрос |
| 201 | Ресурс успешно создан |
| 404 | Ресурс не найден |
| 409 | Конфликт (сотрудник уже существует) |
| 500 | Внутренняя ошибка сервера |

**Формат ошибки:**

```json
{
  "message": "Описание ошибки",
  "stack": "Стек вызовов (только в development)"
}
```
